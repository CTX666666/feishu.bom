# BOM交付守门人：数据安全、风险控制与迭代规划

## 1. 文件目的与当前状态

本文件说明 MVP 拟采用的数据安全、飞书权限、弱网容错、风险治理和分阶段建设路径。当前材料及 Demo 使用人工构造的模拟数据，不包含真实客户、真实人员、真实价格、真实物流和真实交付结果。下述安全控制和飞书能力属于设计方案或待验证项，不能表述为已经完成生产部署。

系统坚持以下权威边界：

- **后端 API、不可变证据账本和规则引擎是业务权威层**，保存最终状态、规则版本、证据链、人工改判及审计记录。
- **飞书是前端工作台和协同编排层**，负责送达、展示、领取、补证、升级、审批或任务协同；飞书消息、多维表格、卡片和流程状态均不能取代后端事实账本。
- AI 只产生候选事实，规则负责确定性门控，人工负责专业判断和例外处理；任何一方均不能绕过证据与审计链。

## 2. 数据分类与最小化

生产接入前拟按以下级别完成数据盘点：

| 数据级别 | 示例 | 处理要求 |
| --- | --- | --- |
| 公开 / 内部 | 公开企业信息、通用操作指引、非敏感枚举 | 按普通内部资料管理 |
| 敏感业务数据 | BOM、物料号、数量、路线、交付时间、供应商、价格 | 按项目和角色隔离；飞书侧只展示完成协同所需的最小字段 |
| 个人信息 | 姓名、手机号、邮箱、地址、飞书用户标识 | 明确处理目的和保留期限；不向无关人员或群聊扩散 |
| 高敏凭证与密钥 | `app_secret`、Encrypt Key、token、签名密钥、对象存储凭证 | 仅存密钥管理服务或受控环境变量；禁止写入源码、日志、卡片和多维表格 |

默认不把完整单证正文和附件复制到飞书消息中。风险卡片拟只展示项目代号、物料摘要、风险类型、截止时间、责任角色、后端 `case_id` 和受控详情链接。访问原始证据时由后端再次鉴权，并使用短期授权、最小下载权限和访问审计。

## 3. 身份、权限与租户边界

### 3.1 最小权限原则

1. 飞书 MVP 拟采用企业自建应用，按实际用例逐项申请 scope；开发期临时权限在发布前移除。
2. 应用身份调用企业级能力时拟使用缓存的 `tenant_access_token`；需要用户授权的操作必须通过 OAuth 获取 `user_access_token`，两者不可混用。
3. 通讯录、审批等敏感权限需管理员审批，并在应用权限说明中列明用途、字段、调用频率和保留期限。
4. 后端不能仅凭“回调来自飞书”即授权业务操作，还需校验用户与本地角色、项目范围、案例状态及动作权限。

### 3.2 建议角色矩阵

| 角色 | 可查看 | 可操作 | 限制 |
| --- | --- | --- | --- |
| 项目经理 | 所负责项目的风险、模块影响、证据摘要 | 分派、升级、发起推演、申请关闭 | 不可修改原始证据和规则版本 |
| 厂区物流 / 采购 | 本责任范围内的厂区与供应事件 | 领取、补充资料、反馈状态 | 不可查看无关项目价格或客户信息 |
| 关务 / 报关人员 | 关务字段、冲突单证和待补项 | 专业复核、提交更正凭证 | 系统不替代其进行 HS 归类或合规结论 |
| 货代 / 运输协同人员 | 被授权航段和事件摘要 | 补充承运人回执、反馈 ETA | 原始 BOM 仅按必要范围披露 |
| 审核 / 管理人员 | 汇总风险、改判和审计记录 | 例外审批、关闭复核 | 高风险例外建议双人复核 |
| 系统管理员 | 应用配置、权限和运行状态 | 配置角色、密钥轮换、审计导出 | 默认无权浏览业务单证正文 |

若后续面向多个企业或外部合作方，必须在数据表、缓存、对象存储路径和日志中强制加入租户及项目边界，不能依赖前端筛选实现隔离。

## 4. 飞书接入安全控制

### 4.1 凭证与网络

- `app_id` 可作为配置项，`app_secret`、Encrypt Key、verification token 和 OAuth 凭证必须进入密钥管理服务，按环境隔离并定期轮换。
- 回调地址必须使用 HTTPS；网关限制请求体大小、速率和允许的内容类型。
- 事件订阅与卡片回调拟使用官方 SDK 完成解密及 verification token / Encrypt Key 校验；验证失败直接拒绝并记录安全事件。
- token 使用 SDK 或后端集中缓存并在过期前刷新；日志只记录 token 指纹或请求追踪号，禁止输出完整 token。

### 4.2 回调与接口安全

- 飞书事件可能重复、延迟或乱序。拟按飞书事件标识及 `case_id + action + case_version` 生成幂等键，并设置合理的去重保留期。
- 卡片按钮提交的是操作意图。后端再次执行用户鉴权、项目授权、案例版本校验和业务规则校验，成功落账后才返回完成状态。
- 对 HTTP 429、网关超时和临时错误使用带抖动的指数退避；对权限不足、参数错误、签名失败不盲目重试。
- 飞书 API 返回 HTTP 成功不代表业务成功，必须检查响应 `code`、记录请求标识，并对非零结果分类处理。
- OAuth 回调必须校验随机 `state`，授权码只使用一次；本地登录会话采用安全 Cookie 或服务端会话，不把长期凭证拼接到前端 URL。

### 4.3 消息与卡片防泄漏

1. 群消息默认不展示价格、客户全称、联系人、地址和完整单证原文。
2. 卡片中的详情链接必须短时有效，并在后端校验当前用户；不能使用永久公开链接。
3. 卡片 `value` 只携带不敏感标识和版本号，不携带密钥、原文、价格或可被篡改后直接生效的业务参数。
4. 多维表格如在 MVP 中启用，仅拟作为协同视图或责任矩阵，不保存权威证据、完整附件和最终计算逻辑。
5. 撤回或删除飞书消息不能等同于删除后端审计记录；两套保留策略需分别定义。

## 5. 证据、AI与审计安全

### 5.1 不可变证据链

每份来源资料拟生成 `source_document_id`、版本和 SHA-256 哈希；每条候选事实生成唯一 `evidence_id`，保留原文位置、抽取值、模型版本、Prompt 版本和置信度。更正、撤销和人工改判以新增记录指向旧记录，不静默覆盖历史。

后端审计日志至少记录：

- 操作者的本地用户标识和关联飞书用户标识；
- 动作前后案例版本、时间、来源渠道和请求追踪号；
- 查看、下载、补证、改判、升级、关闭及权限变更；
- 规则版本、窗口参数版本、AI 模型与 Prompt 版本；
- 失败原因、重试次数和最终处置结果。

### 5.2 AI安全边界

- AI 只抽取原文明确支持的事实，必须区分实际、计划、预测、请求、更正、撤销和未知。
- 模型只能从后端提供的候选 BOM 行中选择，不能编造物料号、BOM 行、文件位置或时间。
- 模型输出必须通过 JSON Schema、主键、数量、单位、时间顺序和证据等级校验后才能进入当前状态视图。
- 低置信度、候选不唯一、单位不可换算、跨单证冲突和高风险例外必须进入人工复核。
- 生产资料调用外部模型前需完成供应商合规评估、数据出境判断、脱敏策略和“不用于训练”等合同约束；无法满足时采用企业受控模型或本地化方案。
- 文档内容可能包含提示注入文本。解析服务应把文档视为不可信数据，不执行其中的指令，不允许模型直接调用写入、审批或通知工具。

## 6. 主要风险与控制措施

| 风险 | 可能后果 | MVP 拟控制措施 | 责任方 |
| --- | --- | --- | --- |
| 计划被误判为实际 | 错误确认装船或到港 | `assertion_mode` 分类；实际状态须满足来源等级和状态机 | AI / 规则负责人 |
| 数量、版本或单位冲突 | 齐套率错误、漏报或重复分配 | BOM 复合主键、数量守恒、版本化单位换算、人工复核 | 数据 / 业务负责人 |
| 飞书卡片重复点击 | 重复派单、重复关闭 | 幂等键、案例版本乐观锁、返回既有结果 | 集成负责人 |
| 飞书或后端暂时不可用 | 消息丢失、案例未处理 | 可靠队列、重试、死信告警、后端独立保存案例 | 平台负责人 |
| 过度授权或群内泄漏 | 商业秘密和个人信息暴露 | 最小 scope、角色隔离、脱敏摘要、短期详情链接 | 安全 / 管理员 |
| 接口沉默被误判为业务停滞 | 错误催办或升级 | 同时检查预期节点、上游条件、源健康和最后更新时间 | 规则负责人 |
| 大模型幻觉或 Prompt 注入 | 虚构字段、绕过规则 | Schema、候选白名单、工具隔离、确定性门控 | AI负责人 |
| 人工无依据改判 | 审计不可追溯 | 强制理由、关闭证据、关键例外双人复核 | 业务负责人 |
| 详情链接被转发 | 越权查看原始资料 | 短期授权、用户绑定、下载水印与访问审计 | 安全负责人 |
| 时区或业务日历错误 | 干预窗口计算偏差 | IANA 时区、统一快照时间、参数版本和固定复算用例 | 规则负责人 |

## 7. 弱网、离线与灾难恢复

跨厂区、港口或海外现场可能存在高延迟和短时断网。MVP 拟采用以下策略：

1. **后端先落账再通知**：业务事件和案例先进入后端权威层，再异步推送飞书，飞书失败不影响事实保存。
2. **可靠出站箱**：业务事务与待发送通知在同一事务中登记，由独立任务投递；成功后保存飞书消息标识，失败进入重试和告警。
3. **客户端动作防重复**：卡片点击携带 `case_id` 和版本号；网络重试时返回同一结果，避免重复关闭。
4. **友好降级**：交互卡片不可用时降级为纯文本摘要和安全链接；详情服务不可用时显示“正在核验/稍后重试”，不伪造完成状态。
5. **缓存只读摘要**：允许读取带“更新时间”和“可能已过期”标记的最近状态；修改、关闭和例外放行必须在线回到权威层校验。
6. **断点续传与哈希核验**：大文件上传使用受控对象存储的分片或断点续传能力；完成后校验哈希，未完成文件不得进入证据链。
7. **备份恢复**：证据元数据、案例、权限映射和审计日志分别制定备份周期、恢复点目标和恢复时间目标，并通过恢复演练验证，而非只验证备份文件存在。

## 8. 监控、告警与验收

### 8.1 运行监控

拟监控以下信号：

- token 获取失败率、缓存命中率和异常刷新次数；
- 飞书 API 成功率、429 频率、响应 `code` 分布和消息送达延迟；
- 回调验签失败、重复率、乱序率、处理耗时和死信数量；
- AI Schema 合法率、字段级准确率、候选 BOM 召回率和人工修订率；
- 规则冲突率、窗口复算一致率、来源健康状态和案例超 SLA 数量；
- 越权访问、异常下载、密钥读取和权限变更等安全事件。

### 8.2 安全验收清单

- [ ] scope 与接口逐项对应，移除无业务用途的权限。
- [ ] `app_secret`、Encrypt Key、token 未出现在源码、前端、日志、卡片或多维表格中。
- [ ] 事件订阅和卡片回调已完成验签、解密、重放与重复推送测试。
- [ ] 不同项目、角色和群聊之间的查看及操作权限已做正反向测试。
- [ ] 卡片篡改、并发点击、旧版本关闭和越权补证均被后端拒绝。
- [ ] 飞书故障、后端超时、数据源异常和弱网场景均有可理解的降级提示。
- [ ] 原始证据、规则结果、人工改判和关闭证据能够串成完整审计链。
- [ ] 数据保留、到期删除、备份恢复和安全事件响应流程已明确责任人。

## 9. 三阶段迭代路线

### 阶段一：当前模拟原型——证明数据关系和核心规则可演示

**目标**：用一个模拟欧洲项目、一版 BOM、一条海运线路，验证 BOM 级数据关系、证据追溯、风险计算和页面交互。

**范围**：

- 当前使用 10 个模拟 BOM 节点、28 条模拟物流事件、29 条模拟证据事实和 5 条模拟干预窗口记录；
- 静态原型展示 BOM 映射、数量守恒、证据等级、状态关系和最迟窗口，AI模型调用与企业接口尚未实装；
- 飞书应用机器人、风险卡片、卡片回调、多维表格、审批和任务均为设计映射，当前不声称已经接入；
- 跑通电控柜 `4 / 4 / 3` 数量冲突、18 小时窗口、延误 3 天推演、更正回执和关闭证据；
- 文档中给出 HTTPS、密钥隔离、验签解密、幂等、最小权限和纯文本降级的试点要求。

**阶段门槛**：四表主外键和窗口公式校验通过，固定演示场景可复算，静态原型核心交互可用；不得以原型结果替代 AI、飞书接口或企业流程测试结果。

### 阶段二：1个月企业试点版——接入受控流程并验证稳定性

**目标**：在获得企业授权和数据脱敏方案后，选择一个受控项目或一条线路开展小范围试点。

**范围**：

- 将验证集扩充至 50 至 100 个模拟 BOM 节点和 15 至 25 份模拟资料；若企业授权，再引入经过脱敏和权限评审的历史资料；
- 与 ERP / WMS / TMS 或受控文件入口建立增量同步和来源健康监控；
- 建立正式用户、角色、项目范围和责任矩阵，按需评估接入飞书任务、审批或多维表格协同视图；
- 增加可靠出站箱、死信处理、SLA 升级、密钥轮换、数据保留和备份恢复演练；
- 建立人工复核台、误报漏报标注和规则版本回归测试；
- 使用真实但受控的历史案例评估查询耗时降幅、异常召回、送达率和案例闭环周期。

**阶段门槛**：权限隔离与安全测试通过；核心 API 和事件处理达到约定可用性；无不可解释的数据丢失；试点业务负责人确认规则结果能够复算，且误报漏报在约定阈值内。

### 阶段三：3个月企业推广版——多项目、多航线与治理运营

**目标**：在试点证据充分后扩展到更多项目、供应商、厂区、口岸和海外交付团队。

**范围**：

- 配置化管理 BOM 字段映射、证据等级、责任矩阵、业务日历、动作耗时和升级策略；
- 完成多租户/多项目隔离、容量规划、灾备、审计报表和安全运营；
- 建立规则灰度发布、回放评测、版本回滚和跨团队变更审批；
- 仅在真实历史数据量、标签质量和合规条件满足后，再评估 ETA 延误预测、清关耗时分布和航线风险评分；
- 对飞书消息噪声进行分级、聚合和频控，避免高风险通知被普通提醒淹没。

**阶段门槛**：通过容量、故障注入、恢复和权限审计；不同项目之间不存在数据串读；所有自动结论可追溯、可复算、可撤销；预测能力必须单独评估，不能以规则结果或小样本演示冒充模型成效。

## 10. 暂不纳入范围

- 不替换 ERP、MES、WMS、TMS 或现有报关、运输系统。
- 不把飞书多维表格、消息卡片或审批记录设为最终事实库。
- 不由 AI 独立执行报关归类、例外放行、运输改签或安装计划调整。
- 不在缺乏真实历史样本时训练或宣称 ETA 预测模型。
- 不把“资料进入后分钟级处理”表述为物理世界事件的实时感知。
- 不把待验证指标、静态 Demo 交互或拟接入飞书能力表述为已实现生产成果。

企业试点的最小接口、统一回执、错误码及可执行验收用例见 [`06_企业试点接口契约与验收清单.md`](06_企业试点接口契约与验收清单.md)。
