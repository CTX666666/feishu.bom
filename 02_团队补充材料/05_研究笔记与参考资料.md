# BOM交付守门人：研究笔记与参考资料

> 核验基准日：2026-07-17。本文合并研究过程、技术选型依据与参考资料。公开来源用于支撑企业背景、赛事场景、文档智能、追溯、供应链控制塔和政策方向，不用于证明本项目已经在企业落地。飞书部分只列本方案拟核验的官方能力类别，不杜撰具体链接、接口可用性或页面状态。

## 1. 研究问题与判断框架

命题表面是“物料进度可见”，研究后可拆成三个更具体的问题：

1. **对象关联**：外部运单、集装箱、包装、单证与事件，如何稳定映射到内部项目、BOM版本、BOM行和设备模块？
2. **事实可信**：邮件、计划、预测、回执和更正资料冲突时，系统如何区分模型置信度与证据权威性，并保留可复核链路？
3. **行动转化**：风险如何从“红黄绿灯”转化为最迟行动时间、责任角色、补充资料、升级路径和关闭证据？

由此形成方案核心判断：控制塔通常回答网络级“发生了什么、货在哪里”，本方案进一步聚焦项目制装备交付中的“哪个BOM节点受影响、依据是什么、最晚何时处理、由谁闭环”。二者是上下游分工，不是互相替代。

## 2. 研究边界与证据纪律

- 企业官网只用于确认公开业务背景，不推导企业已部署何种物流平台、真实数据质量、痛点频次或损失金额。
- 赛事页面只用于确认赛事和命题入口；最终提交字段、格式、时间与资格要求应以当期官方页面为准。
- 厂商文档用于说明技术能力类别和行业实践方向，不作为本项目已经接入或取得成效的证据。
- 论文和预印本用于说明研究问题与技术路线，不替代真实业务样本上的验证。
- Demo及CSV均为模拟数据和关系样例，不代表企业真实项目、客户、供应商、物流或实施结果。
- 所有准确率、召回率、时延、效率和成本改善均写为待验证目标；不虚构访谈、工单、NPS、内部财务或用户采用事实。

## 4. 业务对象与数据模型笔记

| 对象 | 推荐关键字段 | 研究结论 |
| --- | --- | --- |
| 项目 | `project_id` | 隔离不同海外交付任务 |
| BOM版本 | `bom_version` | 防止变更前后的物料和需求时间串账 |
| BOM行 | `bom_line_id` | 唯一定位物料在特定版本和模块中的位置 |
| 包装 / 集装箱 | `package_no`, `container_no` | 显式表达一料多箱和一箱多料 |
| 物流事件 | `event_type`, `event_time`, `received_time` | 分离业务发生时间与系统获知时间 |
| 来源文档 | `source_document_id`, `document_version`, `hash` | 保存不可变来源和版本关系 |
| 证据事实 | `evidence_id`, `original_text`, `location` | 一份文档产生多条可定位、可复核事实 |
| 干预案例 | `case_id`, `latest_action_time`, `case_status` | 将风险转化为行动并记录关闭链路 |

核心关联键采用 `project_id + bom_version + bom_line_id`。物流事件用 `source_document_id` 连接来源文档，证据事实用 `related_event_id` 回指事件；更正、撤销或改判以新增记录连接前序记录，不覆盖历史。

## 5. 技术路线与选型笔记

### 5.1 混合路线

MVP采用“文档智能 / 大模型 + 确定性规则 + 人工复核”的组合：

1. OCR与文档智能：识别版面、表格、键值对、页码和坐标；
2. 大模型：理解多语言文本，分类事实、计划、预测、催办、更正和未知，完成字段及别名归一化；
3. 规则引擎：执行主键匹配、数量守恒、证据等级、状态机、沉默异常和窗口公式；
4. 人工复核：处理候选不唯一、低置信度、证据冲突、单位无法换算和规则未覆盖场景。

这一分工让AI负责处理非结构化表达，让可审计规则负责业务结论。Prompt只输出候选事实和不确定性，不直接判断“已经交付”。

### 5.2 为什么MVP不训练ETA预测模型

可靠的ETA或清关耗时预测需要足量、可比且有标签的真实历史数据，至少覆盖航线、承运人、口岸、季节、节点定义、异常原因和数据缺失机制。当前模拟样本只能验证数据关系、抽取、规则和交互，不能支持模型训练或外推。因此，MVP采用参数化窗口和反事实推演；积累真实历史数据后，再单独评估预测模型的必要性、基线和增益。

### 5.3 数量守恒

按同一项目、BOM版本和BOM行聚合，建立可解释的数量链：

```text
BOM需求量 -> 已确认出库量 -> 已确认装箱量 -> 发票量 -> 报关量 -> 已确认装船量 -> 签收量
```

部分装运允许各批次之和小于需求量，但不能重复分配或超过需求量。单位不一致时必须通过版本化换算表转换；无法换算、证据冲突或更正未完成时进入人工复核。

### 5.4 证据等级与模型置信度

模型置信度回答“抽取得稳不稳定”，证据等级回答“来源能否证明业务事实”。例如，高置信度识别出的“预计装船”仍是计划，不等同于实际提单或承运人实际事件。状态至少区分：

- `planned`：计划或预计；
- `predicted`：模型或承运人预测；
- `confirmed`：高可信事实源确认；
- `conflict`：数量、时间、版本或来源不一致；
- `unknown`：尚无足够证据；
- `corrected`：更正资料改变前序结论；
- `revoked`：事件被撤销或发生退运。

### 5.5 沉默异常

“下一事件没有出现”可能是业务停滞，也可能是上游尚未完成、同步延迟或接口故障。系统应同时读取预期事件时间、上游条件、数据源最后更新时间和健康状态：只有节点超时、上游条件满足且来源正常时，才标记业务停滞；来源异常时标记“来源待恢复”。

### 5.6 最迟干预窗口

```text
latest_action_time = need_by_time
                   - remaining_transport_hours
                   - action_lead_hours
                   - safety_buffer_hours

remaining_hours = latest_action_time - snapshot_time
```

每次计算保存快照时间、时区、业务日历、参数版本和规则版本，使结果可复算。窗口小于零只表示常规方案已无法守住当前节点，需要评估加急、拆分关键件或调整安装计划，不自动选择业务动作。

## 6. 飞书官方能力类别与拟用方式

以下是实施前应从飞书官方产品说明、开放平台文档、企业管理员配置和赛事提供环境中逐项核验的能力类别。此处不声称所有能力均已开通，也不固化未经核验的接口名称、授权范围或版本。

| 官方能力类别 | 本方案拟用方式 | 实施前必须确认 |
| --- | --- | --- |
| 多维表格与数据协作 | 承载风险清单、责任人、SLA、复核状态、参数表和演示看板 | 数据量、字段类型、关联关系、权限和自动化上限 |
| 文档、云空间与知识协作 | 保存说明文档、操作手册、复盘和经批准的证据索引 | 文件权限、外链策略、版本和保留期限 |
| 消息、群组与机器人 | 推送高风险卡片、提醒责任人、升级超时案例 | 机器人权限、消息卡片交互、频率控制和防骚扰策略 |
| 审批、任务或流程自动化 | 承载人工确认、异常升级、补充资料和案例关闭 | 实际可用产品形态、节点能力、撤回与审计机制 |
| 开放平台、API与事件订阅 | 接收或写回经过授权的数据和状态变化 | 应用类型、授权主体、接口配额、事件可靠性和重试 |
| 身份、组织、角色与权限 | 按项目、岗位和敏感级别限制访问及操作 | 通讯录授权、最小权限、离职与项目结束后的回收 |
| 安全、审计与企业管理 | 配置访问日志、数据分类、保留、导出和管理员控制 | 企业版本、合规要求、日志可见范围和留存政策 |
| AI与文档处理相关能力 | 辅助摘要、信息提取或知识检索，具体使用方式另行验证 | 数据是否出域、模型能力、配额、准确性和人工复核要求 |

建议先用飞书承担“人机协同与案例闭环”，把BOM匹配、数量守恒、证据等级和窗口计算保留在独立、可测试的规则服务中。这样既能利用现有协同入口，也避免把核心业务判断绑定到单一界面或自动化配置。

## 7. 数据治理与合规笔记

生产资料可能包含联系人、地址、价格、客户名称、合同信息、设备结构和商业机密。正式接入前至少完成：

- 数据分类分级、最小化采集和字段脱敏；
- 项目级与角色级权限、最小授权和定期回收；
- 来源文档哈希、版本、访问日志和改判记录；
- 模型调用的数据边界、供应商评估和人工复核机制；
- 数据保留期限、归档、删除及审计导出流程；
- 对外催办内容的审批、敏感字段检查和发送留痕。

涉及报关、HS编码、贸易合规或合同判断时，系统只提供资料核对和风险提示，不替代具备授权与资质的专业人员作出结论。

## 8. 验证设计与研究空白

### 8.1 MVP待验证目标

| 指标 | 目标 | 证据要求 |
| --- | ---: | --- |
| 关键物料与BOM映射准确率 | ≥95% | 与双人复核后的固定标注集比较 |
| 预置异常识别召回率 | ≥90% | 注入数量、时间、版本和缺证异常 |
| 固定场景窗口计算一致率 | 100% | 固定参数后与人工复算比较 |
| AI结论原文证据可追溯率 | 100% | 可回到文件、页码、表格位置或邮件段落 |
| 新资料处理时延 | ≤5分钟 | 比较接收时间与风险输出时间 |
| 查询与核对耗时降幅 | ≥70% | 同任务、同口径的人工流程与辅助流程计时 |

这些数字是验收目标，不是实际效果。样例CSV用于展示数据关系和固定剧本，不构成正式验收集。

### 8.2 尚需企业或赛事环境验证的问题

1. 真实BOM版本、物料别名、包装关系和单位换算的完整度；
2. ERP、WMS、TMS、承运人和邮件等来源的接口、更新频率与权限；
3. 单证版式、多语言分布、扫描质量和更正流程；
4. 不同航线、节点和动作的时长参数及业务日历；
5. 风险责任矩阵、SLA、升级路径和关闭证据标准；
6. 飞书参赛环境或企业环境实际开放的能力、配额与安全要求；
7. 当前人工核对耗时、异常基线、延期及加急成本的可核验口径；
8. 用户对误报、人工复核和通知频率的可接受阈值。

未完成上述验证前，不应扩大自动写回、外部通知或财务收益承诺。
