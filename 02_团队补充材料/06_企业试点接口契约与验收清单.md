# BOM交付守门人：企业试点接口契约与验收清单

## 1. 文档定位

本文把方案落到可联调、可验收的最小接口与试点门槛。当前仓库尚未包含生产后端，也未连接明珞系统或真实飞书租户；以下内容是企业试点参考契约，不是已经上线的接口声明。正式实施时应由企业 IT、业务、安全和数据责任人共同确认字段、权限、网络区域、保留期限及验收数据集。

## 2. 试点部署边界

```mermaid
flowchart LR
    D["ERP / MES / WMS / TMS、关务、承运人、受控文件入口"] --> G["接入网关"]
    G --> Q["原始事件与证据收件箱"]
    Q --> A["AI候选事实抽取"]
    A --> R["BOM映射、证据等级、数量守恒、状态机、窗口计算"]
    R --> C["案例服务与审计账本"]
    C --> F["飞书机器人、AI解释、消息卡片、协同视图"]
    F --> C
```

后端是事实与规则权威层。飞书只读取后端快照并提交协同意图。生产交易继续由原系统完成，原始单证优先保存在企业受控对象存储中。

## 3. 最小接口集合

| 接口 | 方法 | 用途 | 权威结果 |
| --- | --- | --- | --- |
| `/api/v1/events` | POST | 接收采购、厂区、关务、海运和现场事件 | 返回接收回执，不直接承诺业务状态已确认 |
| `/api/v1/evidence` | POST | 登记已扫描文件的版本、哈希、位置和候选事实 | 返回 `evidence_id` 与人工复核状态 |
| `/api/v1/cases/{case_id}` | GET | 查询状态、模块影响、证据摘要、窗口和版本 | 后端案例快照 |
| `/api/v1/cases/{case_id}/actions` | POST | 领取、补证、升级、改判或申请关闭 | 权限、版本和规则校验后的动作回执 |
| `/integrations/feishu/events` | POST | 接收飞书事件订阅与卡片回调 | 验签、解密、去重后进入可靠队列 |
| `/health/ready` | GET | 检查数据库、队列、对象存储和规则版本是否就绪 | 仅供运维与发布门禁，不暴露业务数据 |

### 3.1 事件写入示例

```json
{
  "schema_version": "1.0",
  "event_id": "EVT-WMS-20260728-001",
  "project_id": "EU-A",
  "bom_version": "BOM-V1",
  "bom_line_id": "BL-1001",
  "material_code": "MAT-ECAB-001",
  "event_type": "factory_outbound",
  "assertion_mode": "confirmed",
  "quantity": { "value": 4, "unit": "set" },
  "event_time": "2026-07-28T09:30:00+08:00",
  "received_time": "2026-07-28T09:31:12+08:00",
  "source_document_id": "SRC-WMS-20260728",
  "source_document_hash": "sha256:replace-with-real-digest"
}
```

写入必须同时满足：调用方身份已认证，项目范围已授权，`project_id + bom_version + bom_line_id` 能定位唯一物料节点，单位可识别，时间格式合法，`event_id` 和幂等键未产生冲突。`planned`、`predicted` 与 `confirmed` 分开保存；更正使用新事件和 `supersedes_event_id`，不覆盖旧记录。

### 3.2 统一动作请求

```json
{
  "action": "close",
  "expected_case_version": 8,
  "reason": "报关数量已更正并完成人工复核",
  "evidence_ids": ["EVD-CORR-001"],
  "client_context": {
    "channel": "feishu_card",
    "message_id": "om_example",
    "card_version": "3"
  }
}
```

服务端从认证上下文获取租户、用户和角色，不信任客户端自行声明的权限。关闭动作必须同时通过项目授权、`case.close` 权限、案例版本、关闭原因和关闭证据校验。飞书卡片收到成功回执后才能显示“已关闭”。

### 3.3 统一回执

```json
{
  "source": "backend_authority",
  "request_id": "REQ-20260809-0009",
  "accepted": true,
  "duplicate": false,
  "persisted": true,
  "case_id": "CASE-1001",
  "case_version": 9,
  "rule_version": "WINDOW-V2",
  "audit_event_id": "AUD-20260809-0010",
  "as_of": "2026-08-09T16:00:00+02:00"
}
```

重复请求应返回第一次的业务回执，不能再次落账。旧版本返回 HTTP 409 和 `STALE_CASE`；无权限返回 403 和 `FORBIDDEN`；字段或证据不完整返回 422；签名失败和过期回调在网关阶段拒绝。

## 4. 接口与数据安全门禁

1. 数据源接口使用企业批准的 mTLS、OAuth 2.0 客户端凭证或专线方案；网络和应用身份同时校验。
2. 飞书回调使用官方 SDK 对原始请求体验签与解密，检查事件时间窗、租户归属和事件唯一标识。
3. 所有写接口要求 `X-Request-Id` 与 `Idempotency-Key`，案例动作同时要求版本条件；数据库以唯一约束兜底。
4. 文件先进入隔离区，完成类型识别、大小限制、恶意内容扫描、真实 SHA-256 和对象版本登记后，才能进入解析队列。
5. 文档文本一律视为不可信数据。模型无权调用关闭、审批、通知或写回工具；所有业务动作由规则和服务端授权完成。
6. 详情链接使用短期授权并绑定当前用户与项目；群卡片不展示价格、客户全称、联系人、地址和完整单证原文。
7. 后端采用收件箱、出站箱、重试、死信、对账和恢复演练，飞书不可用时不丢失事实与案例。
8. 日志记录请求号、操作者、项目、动作、前后版本、规则版本和结果，但不记录 token、密钥、完整个人信息或完整单证正文。

## 5. 飞书协同契约

| 用户动作 | 飞书负责 | 后端负责 | 失败时显示 |
| --- | --- | --- | --- |
| AI解释 | 发送查询意图并展示带来源编号的解释 | 返回当前快照、证据摘要、事实时间和规则版本 | “结论正在核验”，不使用旧内容冒充最新结论 |
| 领取 | 提交 `case_id` 与卡片版本 | 校验项目权限、案例状态和幂等键 | 无权限或已被领取时给出明确原因 |
| 补证 | 打开受控上传入口或提交文件索引 | 扫描文件、登记哈希、AI抽取并进入人工复核 | 文件不合规时不进入证据链 |
| 升级 | 收集原因、目标角色和期限 | 校验升级规则并写入审计账本 | 返回可执行的修正提示 |
| 关闭申请 | 收集原因和证据编号 | 校验权限、案例版本、原因和关闭证据 | 403、409或422原因，不显示伪成功 |

## 6. 企业试点推进计划

| 阶段 | 建议周期 | 输入 | 交付物 | 进入下一阶段的门槛 |
| --- | --- | --- | --- | --- |
| 数据体检 | 3-5个工作日 | 1版BOM、字段样例、责任矩阵、单证目录 | 字段映射表、质量问题单、脱敏方案 | 复合主键、单位、时间和来源能解释 |
| 影子运行 | 2周 | 文件旁路或只读接口 | 固定验收集、差异报告、规则版本 | 关键准确性与可追溯指标达到阈值 |
| 受控飞书协同 | 1-2周 | 企业自建应用、最小scope、试点用户 | 机器人、卡片回调、权限矩阵、故障降级 | 安全测试、幂等和完整闭环通过 |
| 30天试点评估 | 30天 | 单项目、单航线、业务对照组 | KPI报告、误报漏报复盘、TCO参数 | Go / No-Go会议书面确认 |

试点默认旁路运行，不中断原流程。只有人工确认后的风险案例和协同状态可受控写回，ERP、MES、WMS、TMS 的生产交易不由本系统修改。

## 7. 验收用例

| 编号 | 验收动作 | 通过标准 |
| --- | --- | --- |
| F01 | 导入已冻结BOM版本 | 复合主键无重复，父子关系、单位和需求时间可解释 |
| F02 | 注入 `4 / 4 / 3` 单证冲突 | 自动阻断状态确认，生成证据编号、受影响模块和18小时窗口 |
| F03 | 运行72小时延误推演 | 得到 `18h -> -54h`，结果标记为只读且不写回案例 |
| F04 | 提交模拟更正回执 | 原 `3 SET` 证据保留，新 `4 SET` 候选进入人工复核 |
| F05 | 人工确认后重算 | 案例版本递增，数量守恒通过，规则版本可回查 |
| F06 | 合法关闭案例 | 权限、版本、原因、证据均通过后才返回权威回执 |
| S01 | 签名错误 | 在网关阶段返回 `INVALID_SIGNATURE`，不进入业务队列 |
| S02 | 过期回调 | 返回 `EVENT_EXPIRED`，不进入业务队列 |
| S03 | 重复事件或重复点击 | 返回第一次回执，不产生第二条业务结果 |
| S04 | 旧案例版本 | 返回 409 和 `STALE_CASE`，要求刷新 |
| S05 | 只读角色申请关闭 | 返回 403 和 `FORBIDDEN`，数据不变 |
| S06 | 文档包含HTML或提示注入 | 按文本显示，模型工具权限为 `none`，不触发动作 |
| R01 | 飞书暂时不可用 | 后端案例仍落账，通知进入出站箱并可补发 |
| R02 | 规则服务超时 | 显示“正在核验”，不使用缓存结果冒充新结论 |
| R03 | 备份恢复演练 | 在约定RPO、RTO内恢复案例、证据元数据、权限和审计链 |

## 8. MVP指标与Go / No-Go

| 指标 | 试点目标 | 证据 |
| --- | ---: | --- |
| 关键物料与BOM行映射准确率 | ≥95% | 双人复核后的冻结标注集 |
| 预置异常召回率 | ≥90% | 数量、时间、版本和缺证注入集 |
| 结论原文证据可追溯率 | 100% | 文件、页码、表格位置或邮件段落 |
| 固定场景窗口复算一致率 | 100% | 固定时钟、时区、参数和规则版本 |
| 新资料进入到风险更新 | ≤5分钟 | 接收时间与风险输出时间 |
| 卡片动作幂等率 | 100% | 重复、乱序和并发回调测试 |
| 跨项目越权 | 0例 | 正向与反向权限测试 |

任何映射准确性、证据追溯、窗口复算或权限隔离未达门槛，均不得进入受控生产试点。效率指标未达标可优化流程；业务问题未被证实则停止扩张，不通过增加功能掩盖证据不足。
