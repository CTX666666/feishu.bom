# BOM交付守门人：AI物料全过程跟踪

> 让跨厂区、跨关务、跨海运的每个关键物料，在影响安装前被看见、被解释、被派给正确的人。

## 真实企业命题

- 赛事：2026 AI先锋未来人才大赛
- 企业：明珞股份
- 赛区：南部赛区（广州）
- 命题：**AI + 物料全过程跟踪：如何让跨厂区、跨关务、跨海运的物料进度实时可见？**

## 60秒了解项目

海外项目交付通常跨越供应商与多个厂区，再经过装箱、申报、放行、装船、到港、签收和现场领用。BOM版本、仓储记录、装箱单、报关资料、货代邮件和承运人事件分散在不同系统与沟通渠道中。普通看板能显示运单或集装箱位置，却难以直接回答某个关键物料是否齐套、会影响哪个设备模块、结论来自什么证据，以及最晚何时必须采取行动。

“BOM交付守门人”以BOM行为业务主键，把跨厂、关务和海运事件还原为可追溯的物料证据链，通过确定性规则识别数量冲突、计划状态误判、数据源沉默和干预窗口风险，再由飞书把风险送达责任人并承载后续协作。

## 方案流程

```mermaid
flowchart LR
    S["ERP/MES/WMS/TMS、关务、承运人、邮件与单证"] --> A["后端API与AI事实抽取"]
    A --> B["BOM行匹配与证据账本"]
    B --> R["数量守恒、状态机、源健康与窗口计算"]
    R --> F["飞书多维表格、机器人、消息卡片与审批"]
    F --> C["认领、补证、升级、关闭与复盘"]
```

## 为什么需要AI

跨境资料包含PDF、表格、邮件和多语言文本，人工需要反复核对物料别名、数量、箱号、地点和时间。AI负责从非结构化资料中抽取可由原文直接支持的候选事实，并区分“计划、预测、实际、催办、更正和撤销”。AI不直接确认业务状态；BOM匹配、证据等级、数量守恒和时间计算由规则完成，冲突与低置信度结果交由人工复核。

## 为什么使用飞书

飞书在本方案中是统一前端工作台和协同编排层，不是核心数据库：

- 多维表格呈现项目、BOM风险、证据摘要和干预案例视图。
- 机器人与AI技能提供风险查询、证据解释、延误推演和催办草稿。
- 交互式消息卡片承载触达、认领、转派、补证和升级。
- 审批承接运输升级、交期调整和人工改判等高影响动作。
- 云文档沉淀干预包、规则说明和案例复盘。

后端API、不可变证据账本和确定性规则仍是业务权威来源。上述飞书能力属于企业试点阶段的拟接入设计，当前原型不冒充已经连接飞书租户。

## 当前完成内容

- 基于模拟数据的BOM物料列表与全过程时间线。
- 跨厂调拨、关务、海运、签收与安装领用事件样例。
- 单证原文、位置、抽取值、置信度和证据等级展示。
- 数量冲突、计划状态、沉默异常和更正链案例。
- 最迟干预窗口与“延误3天”反事实推演。
- 干预包、飞书AI解释卡片、模拟补证、人工复核、规则重算和受控关闭完整闭环。
- 签名错误、过期回调、重复事件、旧版本、无权限角色和不可信文档指令六类安全门禁。

## 数据与成果边界

- 所有CSV均为人工构造的模拟数据，不包含明珞股份真实项目、客户、人员、物流、价格或交付结果。
- 当前样本用于方案说明、静态原型和数据关系核验，不用于模型训练或统计外推。
- 95%映射准确率、90%异常召回率、70%核对耗时降幅等均为MVP待验证目标，不是已经实现的企业成效。
- “实时”指资料或API事件进入系统后的分钟级处理目标，不表示已部署物理世界实时感知设备。
